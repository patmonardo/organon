# Collections Dataset: Feature Structures and Model Preparation

File: `gds/examples/dataset_feature_structures.rs`

## Principle

This exemplar teaches **feature structures as the symbolic algebra** for semantic marking.
Feature structures are the threshold where observation becomes essence. You learn how:
- Features are typed, addressed marks (not anonymous columns)
- Unification prepares entities for Reflection
- Modality stamps capture execution decisions
- Ontology image realization records what happened

This exemplar shows the **three-box form processor**: Box 1 (prepare essence), Box 2 (lower
features), Box 3 (realize images). This is where Observation touches Reflection.

## What It Does

1. Constructs feature structures from parsed trees
2. Performs unification to prepare entities for semantic marking
3. Stamps modality on features (necessity, possibility, contradictorian)
4. Executes feature projection into DataFrame expressions
5. Realizes ontology image tables: the record of what was marked and how

The example is dense. It shows the machinery of semantic preparation. You are seeing
the bridge between syntax and essence.

## The Arc

**Threshold between Stage 1 (Observation) and Stage 3 (Reflection)**: The form processor interior.

This exemplar is special because it shows the invisible machinery. The three boxes are:

1. **Essence Preparation** (Box 1): Feature structures are parsed and unified
2. **Feature Execution** (Box 2): Marked features become DataFrame expressions
3. **Image Realization** (Box 3): Ontology tables record which marks were executed and what modality they carried

When this exemplar completes, the entity has been marked but not yet reflected. It has
essence stamps but not the full seven-moment arc. That comes in the next exemplar.

## Key Vocabulary

- **Feature Structure (FeatStruct)**: a typed, reentrant, unifiable symbolic structure. See [feature-structures.md](../../REFERENCES/dataset/feature-structures.md)
- **Unification**: the symbolic operation that stamps features with their values. See [unification.md](../../REFERENCES/dataset/unification.md)
- **Modality**: the execution decision (necessary, possible, contradictorian) on a feature. See [modality.md](../../REFERENCES/dataset/modality.md)
- **Ontology Image**: the realized record of marks and modality. See [ontology-images.md](../../REFERENCES/dataset/ontology-images.md)

## Notes for Students

**Watch for**: This exemplar is syntactically dense. The feature structures are written in
Scheme-like notation. Take time to parse the examples. Each feature structure tells a story
about what semantic marks are being applied.

**Key insight**: Feature structures are not new data. They are a _view_ over the parsed tree
that emphasizes the semantic roles rather than syntactic structure. The unification operation
ensures that roles are properly stamped before execution.

**Try this**: Write a simple feature structure. Unify it. See the resulting modality stamps.
Add it to the ontology image table. This is the core operation of essence preparation.

**Critical**: Modality is not optional. Every feature that gets executed has a modality.
This record is the basis of absolute interpretability.
