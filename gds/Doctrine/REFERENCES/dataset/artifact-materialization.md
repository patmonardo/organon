# Artifact Materialization

Materialization turns compiled intent into Dataset artifact tables.

---

## Doctrine

Materialization is where the program becomes inspectable. It should emit records, relations, properties, and provenance rather than hiding results in opaque structures.

---

## Core Tables

- artifact records
- artifact relations
- artifact properties
- provenance records

---

## Review Rule

If a materialization path cannot be inspected as tables, it is not yet doctrinally complete.
