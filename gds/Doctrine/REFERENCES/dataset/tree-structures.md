# Tree Structures

Trees are observed structure: syntax, markup, parse shape, dependency shape, or document hierarchy made inspectable.

---

## Doctrine

Trees belong to Observation. They organize appearance so marks and Reflection can operate later.

---

## ParseValue

A parse value is the typed content of a node: tag, constituent label, dependency role, token, literal, or structural marker.

---

## Parse Forest

A parse forest is a set of possible trees. It preserves ambiguity until later stages resolve it.

---

## Review Rule

A tree artifact should preserve node identity, parent relation, span or position, label, and source provenance.
