# Morph as Whole of Cognition

## Morph as ContextualizedShape

**A Morph as a ContextualizedShape is really for us the point at which we can Apply our Processor to Appearances.**

- **Morph** = ContextualizedShape (Shape + Context)
- This is the point where we can **Apply our Processor to Appearances**
- Morph enables application to Entity/Property/Aspect

## Morph as Whole of Cognition

**It is a Presupposition Chain. We can enter into Entity/Property/Aspect unless we have a Morph as a type of Whole of Cognition.**

- **Morph** = A type of Whole of Cognition
- **Presupposition Chain**: We need Morph before we can enter into Entity/Property/Aspect
- Morph is the **presupposition** for the contained pipeline

## The Architecture

### Container Pipeline (Principle)
```
Shape → Context → Morph (ContextualizedShape)
```

### Contained Pipeline (Appearances)
```
Entity → Property → Aspect
```

### The Presupposition Chain

**We cannot enter into Entity/Property/Aspect unless we have Morph as a type of Whole of Cognition.**

- **Morph** = Whole of Cognition (Shape + Context)
- **Morph** = Presupposition for Entity/Property/Aspect
- **Morph** = Point at which we can Apply Processor to Appearances

**However**: Entity doesn't emerge from Shape directly - it emerges by applying Shape to Appearances. And we might be able to have Entity emerge from Shapes without Morph? Maybe/maybe not. Some "Science" is possible without the Whole of Cognition, but complete Science might require Morph.
