# Idea Workbook Process

Dual-track process for building Idea workbooks as both principle network and philosophical education.

## Tracks

- Track A (`Kernel`): extract top-level logical principles, constraints, and transitions.
- Track B (`Pedagogy`): explain why each move matters, including historical and conceptual framing.

## Pass Sequence

1. Segmentation pass: split source text into coherent argument blocks.
2. Kernel pass: encode principle nodes into `IDEA-PRINCIPLES-LEDGER.md`.
3. Pedagogy pass: write teaching notes linked to Kernel IDs.
4. Reconciliation pass: ensure pedagogy does not weaken logical claims.
5. Table pass: update `IDEA-KEY-TABLES.md` and cross-links.

## Entry Contract (minimal)

Each workbook entry should include:

- `Entry ID`
- `Scope` (chapter/section)
- `Kernel IDs`
- `Claims`
- `Transitions`
- `Pedagogy Notes`
- `Evidence Anchors`

## ID Policy

- Keep IDs stable after first use.
- Never recycle deleted IDs.
- New IDs append sequence values only.

## Relation Vocabulary

Use only:

- `supports`
- `contrasts`
- `negates`
- `sublates`
- `presupposes`
- `refines`
- `transitions_to`

## Scope Rule

- Ledger is canonical for principle identity.
- Workbook pages are canonical for exposition and evidence.
- Tables are canonical for cross-section navigation and synthesis.
